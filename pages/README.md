# Public pages

Active public route files remain at the repository root so existing URLs and GitHub/Vercel
links do not change. Their implementation is organized here conceptually:

- Entry: `index.html`, `visitor.html`
- Content: `article.html`, `case.html`, `video.html`, `thought.html`, `content.html`
- Browse: `article-list.html`, `case-list.html`, `video-list.html`, category/topic pages
- Media: `gallery.html`
- Information: `about.html`

Page-specific JavaScript lives in `scripts/pages/`. Page-specific CSS lives in
`assets/styles/pages/`.
